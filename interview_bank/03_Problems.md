# 对话知识点浓缩

## 1. 学习与表达方法
- 记忆框架：定义 -> 术语解释 -> 例子 -> 面试表达。
- 面试策略：先给 30 秒短答，再补项目化细节。
- 排错/优化原则：先测量再结论，避免“凭感觉优化”。

## 2. C++ 资源管理与对象语义
- RAII：把资源生命周期绑定对象生命周期，构造获取，析构释放。
- 不用 RAII 的风险：泄漏、忘记释放、异常路径清理不全。
- 智能指针：
- `unique_ptr`：独占所有权，默认首选。
- `shared_ptr`：共享所有权。
- `weak_ptr`：打破共享环，避免循环引用泄漏。
- 内存问题：
- 泄漏：分配后不可达且未释放。
- 悬垂：指向已释放内存（典型是 use-after-free）。
- 定位思路：ASan -> 最小复现 -> 回溯分配点。

## 3. 容器与性能
- `vector` 扩容：容量不足会重分配并迁移元素。
- `size` vs `capacity`：逻辑大小 vs 预留容量。
- 扩容副作用：迭代器/引用失效。
- 性能实践：已知规模先 `reserve`，关注移动构造代价。

## 4. 值类别、移动语义、转发
- 左值/右值：是否有持久身份与可取址特性。
- 移动语义：转移资源所有权，减少深拷贝。
- 完美转发：`T&& + std::forward<T>` 保留值类别。
- 不完美转发后果：右值退化成左值，错失移动机会，甚至影响可编译性（move-only 类型）。

## 5. 多态与类型转换
- vtable/vptr：运行时多态通过虚表动态分派。
- 多态开销：间接跳转、内联困难，小函数高频调用更敏感。
- `const_cast`：仅在确认底层对象可写时谨慎使用。

## 6. 并发基础（C++）
- 常见风险：数据竞争、死锁、活锁、ABA、可见性问题。
- 死锁根因：多锁加锁顺序不一致（A->B 与 B->A）。
- 无锁优化：不是“完全不用锁”，而是减少阻塞点和竞争。
- 原子操作：单步不可分割，对并发计数等简单共享状态有效。
- 伪共享（False Sharing）：不同变量落在同一缓存行，导致缓存抖动和吞吐下降。

## 7. 模板与工程权衡
- 价值：复用、类型安全、零开销抽象潜力。
- 成本：编译时间增加、代码膨胀（code bloat）。
- 实践：公共模板层保持薄，复杂逻辑下沉到非模板实现。

## 8. 错误处理策略
- 异常：适合错误传播链长、内部逻辑希望更干净。
- 错误码：适合跨边界（跨语言/跨进程/公共 API）和显式控制。
- 关键原则：同一模块统一风格，避免混用导致调用方困惑。

## 9. 可测试性设计（C++）
- 可测试三要素：可替换依赖、可控输入、可观察输出。
- 依赖注入（DI）：外部依赖作为参数传入，不在内部硬编码创建。
- 典型例子：`Clock` + `FakeClock` 测超时逻辑。
- 结论：核心逻辑纯化，IO/网络边缘化，测试稳定且成本低。

## 10. Rust 核心认知
- Rust 不复制 C++ 的“完美转发问题”，以所有权/借用模型规避大量类别错误。
- `Box<T>`：堆上独占所有权；常用于递归类型和大对象搬移。
- `Rc<T>`：单线程共享所有权（引用计数）。
- `Arc<T>`：多线程共享所有权（原子引用计数）。
- `Send`：值可在线程间转移。
- `Sync`：`&T` 可在线程间共享。
- Future 模型：Future 是可轮询状态机；executor 调度，reactor 通知 IO 就绪。
- 异步禁忌：偷偷阻塞会拖垮 runtime（如应放到 `spawn_blocking`）。

## 11. Rust 性能与优化
- profiling：测热点函数、分配行为、等待时间，不靠猜。
- flamegraph：用调用栈宽度表示耗时占比，优先优化最宽热点。
- 优化闭环：先测量 -> 找热点 -> 小步优化 -> 回归验证。

## 12. Rust 在高并发后端的取舍
- 优势：内存安全、并发安全、性能稳定、尾延迟表现常更稳。
- 局限：学习曲线陡、团队不熟时开发效率门槛较高。
- 选型原则：看业务延迟目标、团队能力、迭代节奏，不做“语言信仰”决策。

## 13. Python 并发与 GIL
- GIL 存在原因：简化解释器内部一致性与内存管理复杂度。
- 结论不是“不能并发”：
- IO 密集型可用多线程并发（等待期间会释放 GIL）。
- CPU 密集型线程并行受限，常用多进程或下沉到 C/Rust。
- 区分概念：并发不等于并行。
